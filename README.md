# ssl-worldcup26-pages

## 项目简介

本仓库用于归档和发布多个独立 HTML 页面。页面内容围绕通用主题整理，不针对任何特定域名或网站。每个页面均为自包含的静态文件，便于浏览、存档和分享。

## 目录结构

```
ssl-worldcup26-pages/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的静态资源（图片、样式、脚本等）
└── README.md       # 本文件
```

- **pages/**：包含所有独立 HTML 页面，文件名以页面用途或编号命名。
- **assets/**：存放页面所需的公共资源，按类型或页面分组。

## 页面归档说明

- 每个页面均为完整 HTML 文档，可直接在浏览器中打开使用。
- 页面之间无依赖关系，可单独部署或迁移。
- 归档遵循以下原则：
  - 不包含外部追踪代码或分析脚本。
  - 不嵌入指向特定网站的链接或推广内容。
  - 样式和脚本尽量内联或存放在 `assets/` 中。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/ssl-worldcup26-pages.git
   ```
2. 打开 `pages/` 目录下的任意 HTML 文件即可查看内容。
3. 如需部署，可将 `pages/` 目录整体上传至静态托管服务。

## 维护说明

- 欢迎提交 Pull Request 添加新的页面或改进现有内容。
- 请确保新页面符合归档原则，不包含外部依赖与营销信息。
- 如发现页面无法正常显示，可提交 Issue 描述问题。
